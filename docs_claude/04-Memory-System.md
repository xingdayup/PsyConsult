# 04 - 记忆系统

## 为什么分两层？

智能系统需要记住两种东西，生命周期完全不同：

| 记住什么 | 有效期 | 例子 |
|----------|--------|------|
| 这次会话说了什么 | 会话结束就忘 | "刚才鉴别的那个患者 PHQ-9 多少分？" |
| 这个患者的临床画像 | 长期保留 | "上次诊断中度抑郁，用舍曲林 50mg，复诊周期 4 周" |

所以用两种存储：Redis 管短期、Milvus 管长期。

## Redis 短期记忆

### 怎么存

```
Redis Key: memory:short:{doctor_id}:{session_id}
Value:     JSON 数组 [{role, content}, ...]
TTL:       30 分钟自动删除
```

每一轮对话结束后，系统把"医生输入 + AI 回复"追加到 Redis。

### 怎么用

下一轮对话开始前，从 Redis 取最近 10 条消息，拼成：

```
【近期对话历史】:
医生: 患者近两周情绪低落、失眠...
CDS: 症状清单: 情绪低落、失眠、兴趣减退、食欲减退。PHQ-9≈14分。
医生: 刚才那个患者，PHQ-9 里第 9 条是多少？
```

CDS 看到上文就知道"那个患者"指谁。

### 两个保护机制

- **TTL 30 分钟**：会话结束半小时后自动删除，不占内存
- **超过 10 条自动裁剪**：只保留系统消息 + 最近 6 条对话，防止无限膨胀

### 降级

Redis 挂了不会崩——`_available` 标志位自动变 `False`，所有操作返回空值。系统照样回答问题，只是丢了短期上下文。

## Milvus 长期记忆

### 和 Redis 的区别

Redis 存的是**原始对话**，靠 key 精确查找。Milvus 存的是**提炼后的临床标签**，靠向量做语义搜索。

### 存什么

每轮对话后台，LLM 分析对话内容，提取：

```
主诉: 情绪低落、失眠 2 周
诊断: 中度抑郁发作 (ICD-11: 6A70)
用药: 舍曲林 50mg qd
PHQ-9: 14 分
复诊周期: 每 4 周
过敏史: 无
```

这些标签被转成向量存入 Milvus，按 `doctor_id` 隔离。

### 怎么用

患者一个月后复诊，医生打开系统。当前问题"上次那个抑郁患者怎么样了"被转成向量，Milvus 搜索到最相关的标签——"诊断: 中度抑郁, 用药: 舍曲林 50mg, PHQ-9: 14"→ 注入 Agent → CDS 自动对比前后 PHQ-9 分数，判断疗效。

### 什么时候触发

- **每 5 轮对话**：后台静默提取一次，不阻塞当前对话
- **会话结束时**：最后一次提取，然后清除 Redis

### 去重

新提取的标签和 Milvus 里已有的做子串匹配——"诊断: 中度抑郁"和"诊断: 中度抑郁发作"判为重复，跳过不重复存。

## 降级和限制

- Redis 或 Milvus 挂了不崩，`_available` 标志位保护
- 目前按 `doctor_id` 隔离——不同医生看不到彼此的诊疗记录
- 多医生会诊共享同一患者数据——当前版本不支持，标记为待扩展
- 提取质量依赖 LLM，可能漏掉关键信息（如过敏史）
