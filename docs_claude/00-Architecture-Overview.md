# 00 - 系统架构全景概览

## 这是什么？

一个**精神科临床决策支持系统**。医生输入患者的症状描述，系统自动完成从症状提取到药物审查的完整推理链。

**定位**：辅助参考工具，所有诊断建议须经执业医师确认。

## 三层架构

```
浏览器 (Vue 3)                    ← 医生交互界面
    │ POST /api/chat
    ▼
FastAPI (Python)                  ← API 网关 + SSE 流式返回
    │
    ▼
LangGraph 多 Agent 引擎 (Python)   ← 核心推理
    ├── 症状提取 Agent            口语→术语 + 量表推断
    ├── 鉴别诊断 Agent            ICD-11 逐条对照
    ├── 治疗推荐 Agent            一线/二线方案
    └── 药物审查 Agent            相互作用风险矩阵

依赖的基础设施:
    Redis     短期记忆（记住本轮对话）
    Milvus    向量检索（ICD-11 + 指南）+ 长期记忆
    Neo4j     知识图谱（疾病-症状-药物关系）
    MySQL     模拟诊疗数据（预留）
```

## 一次完整请求走一遍

```
医生输入 "患者近两周情绪低落、失眠、食欲下降..."

1. FastAPI 收到 POST /api/chat
       │
2. 检查语义缓存（Milvus）
   └─ 未命中，继续
       │
3. 加载记忆上下文
   ├── Redis: 本轮对话历史（"刚才说到哪了"）
   └── Milvus: 患者历史临床画像（上次诊断、用药）
       │
4. Orchestrator 路由 → "symptom_extraction"
       │
5. 症状提取 Agent
   ├── "睡不着" → 映射到标准术语 "失眠"（ICD-11: MB21.0）
   ├── 推断 PHQ-9 各条目得分 → 总分约 14 分（中重度）
   └── 输出结构化症状 JSON
       │
6. 鉴别诊断 Agent（自动衔接，不需要再路由）
   ├── 用症状清单搜索 Neo4j 知识图谱 → 匹配到"抑郁发作"和"双相II型"
   ├── 用 Milvus 检索 ICD-11 诊断标准原文
   ├── 逐条对照: ✅情绪低落 ✅兴趣减退 ✅睡眠障碍 ✅食欲减退 ❌自责自罪
   └── 结论: 符合抑郁发作 (ICD-11: 6A70)，排除双相II型
       │
7. 治疗推荐 Agent（自动衔接）
   ├── 检索中国抑郁障碍防治指南（Milvus RAG）
   ├── 查知识图谱中的 SSRI 药物列表
   └── 输出: 一线 → 舍曲林 50mg/d，二线 → 换文拉法辛或联合 CBT
       │
8. 药物审查 Agent（自动衔接）
   └── 只有一种药物，无相互作用风险
       │
9. SSE 流式返回完整报告给浏览器
```

## 技术栈一览

| 层 | 技术 | 在这个项目里干什么 |
|----|------|-------------------|
| 前端 | Vue 3 + Element Plus | 医生聊天界面，Markdown 渲染诊断报告 |
| API | FastAPI + SSE | 接收请求，流式返回结果 |
| 编排 | LangGraph (StateGraph) | 管理 4 个 Agent 的执行顺序和状态传递 |
| LLM | DashScope (Qwen-Plus) | 所有推理——路由、提取、诊断、推荐 |
| 向量库 | Milvus | ICD-11 指南检索 + 长期记忆 |
| 图库 | Neo4j | 疾病-症状-药物-副作用的关系网络 |
| 短期记忆 | Redis | 记住本轮对话上下文 |

## 关键设计决策

**固定流水线而非动态路由**：临床推理的步骤是确定的——必须先有症状才能诊断，必须先有诊断才能推荐治疗。所以 4 个 Agent 用固定顺序串起来，不需要复杂的分支判断。

**ICD-11 而非 DSM-5**：ICD-11 由 WHO 发布，免版权，诊断标准可以完整放入系统。DSM-5 有版权限制。

**量表从对话推断**：PHQ-9/GAD-7 不需要医生手动录入 9 个题目的分数，AI 根据症状描述强度自动推断。牺牲一定精度换取易用性。

**知识图谱 + 向量检索双 RAG**：精确的结构化关系（"舍曲林有哪些副作用"）走图谱，大段的文本描述（"指南中维持期治疗的建议"）走向量检索。各做各擅长的事。
